How to write an Eclipse documentation

Java Documentation

Like very good developers we are, we provide a Java-doc of our plug-in classes. You can found this ATL Java-doc here.

Creating a documentation plug-in

Eclipse provides an example source and a fully documentation here.

Firstable you have to create a new simple plug-in project. The only thing you have to do is to add the "org.eclipse.help.toc" extension point. Then you can specify the primary table of contents file from which the help will beginning the display. After that, if you know how to write HTML web pages, you just have to write them and make them refer inside an XML toc file.

The XML files

It's suggested to split the XML table of contents file into one for each main topic. This is a more sophisticated way to update the documentation porject.

Refering an other page

It's very easy to understand at the first time out to refer to an other Eclipse documentation page or a Java-doc page. This table shows you the tested value for this kind of references :

Tags from the CSS

As we want to create XHTML style web pages, we have redefined some tags through a style sheet.

To match witch has already been done in other Eclipse documentations, we recommend you to use the tags with the only attribute "class". This will cause to every craetes similar pages. We propose you to put a main title with the "h1" tag, which has to be report in the Eclipse XML toc file as the label of a topic. Below, you can put sub-topic with the "h2" tag, and this has to be in a sub-topic in the XML file.
