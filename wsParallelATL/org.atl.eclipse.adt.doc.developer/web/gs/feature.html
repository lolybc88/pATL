What is a feature?

A way to install new eclipse plug-ins is to manually unzip the corresponding zip file into the directory where eclipse is installed. Doing so is not convenient due to many reasons among which the fact that software are, most of the time, made up of several plug-ins. Installing a new software product will then require to install several plug-ins and this could become really not convenient if we would just update one of the plug-ins.

That is why eclipse proposes a simple and efficient mechanism for installing and updating eclipse software product based on the concepts of feature and update site.

A feature is a way of grouping and describing plug-ins that comprise a product. Grouping plug-ins into features and publishing them on to an update site allows the product to be installed and updated easily.

By using the eclipse update manager, only specify the URL of the update site and eclipse will Automatically search for features available on this update site and you can choose what features to install. You can also use the update manager to search for updates of already existing features.

Creating a feature

So as to create a feature, just do the following:

  • Select File > New > Feature Project and fill in the requested information.

Eclipse will create a file named 'feature.xml' in the project folder. Double clicking on this file will open a user-friendly editor that will allow to control (add/remove plugins, ') the content of the feature.

Creating an update site

Once the feature created, it is now possible to create an update site by doing the following:

  • Select File > New > Other ;
  • Then in the Plug-in Development folder select Update Site Project ;
  • Then fill in all the requested information.

It is possible to generate a sample web page by ticking the check box 'generate a sample web page'.

After that Eclipse would have created a file named 'site.xml'. Double clicking on this file will open a user friendly interface that will allow to control (add/remove/publish features, ') features.

Building a feature generates one jar file per plug-in involved in this feature and puts the jars in the 'plug-ins' directory. The features are localized in the 'features' directory. Publishing a feature makes it available from the update site. It should be pointed out that a feature cannot not be installable from the update manager until it is published, even if it has been added to the update site and built.

Managing feature versioning

One interesting thing with features is that they can be versioned. Indeed, if you have a feature made up of three plug-ins, if you fix a bug or add some functionality on one or more of the plug-ins you can keep on the former feature and easily create a new one with the same name but a different version. And both can made be available through the update site. The question is how to do that?

  • Change the feature version by editing its 'feature.xml' file ;
  • Remove the involved plug-ins from the feature ;
  • Change the involved plug-ins version by editing their 'plugin.xml' file ;
  • Save all the changed "plugin.xml" files ;
  • Add the removed plug-ins (now they have a different version) ;
  • Save the feature ;
  • Edit the 'site.xml' file of the update site and click on 'Add' in the 'features' tab ;
  • Select the required feature (the one with the same name than the former but with a different version) ;
  • Build this feature ;
  • Publish it to make it available through the update manager.

note: when removing add adding plugins, you can add or remove any plugin of your choice.

Using the update manager

To open the update manager only select Help > Software Updates > Find and Install'
Then two possibilities are offered:

  • Search for updates of the currently installed features. It should be highlighted that this functionality will work only with features that specify an update URL. To specify an update URL for your feature, only edit its 'feature.xml' and in the 'overview' tab, right click on 'Update URLs' (it is on the right) and select New > Update URL then type in the URL of the update site where you put the new feature (a feature is considered an update of another if it has the same name but a different version) ;
  • Search for new features to install. You can search on already existing update sites and if none click on 'Add Update Site' and type in its URL to add it. Then choose an update site and click next. You will be listed all the features available on that selected update site (excepted those already installed). You can then select which feature to install'.

For further information about features and update sites see the eclipse help and article: http://www.eclipse.org/articles/Article-Update/keeping-up-to-date.html
