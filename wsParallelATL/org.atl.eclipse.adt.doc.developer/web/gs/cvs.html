Team programming with CVS

When you have to work in a group of people, a good way to work with concurrent versions is to use CVS. Eclipse allows this by providing a CVS sharing. Lots of informations in this page can be also found in the Eclipse CVS help page.

Sharing a new project using CVS

To share a CVS repository, we recommend you to do this :

  1. Go into the CVS Repository Exploring perspective, and create there a new repository location with these values to save it :
    VariableValue
    Host172.16.9.33
    Repository path/var/lib/cvs
    User<your username>
    Password<your password>
  2. Create a simple project using the usual Eclipse new project wizard ;
  3. The next step is to share the project. To do so, right click on the project, and select Team > Share project ;
  4. You could now use the repository location created before to share your project ;
  5. Now you need to specify which module name to use. Because if you tell to Eclipse to use the project name, it will create a bad directory on the CVS, and this is not a good idea ! Fill the gap using the path as on the CVS (org.atl.eclipse/adt/editor) instead of the project ID or name (org.atl.eclipse.adt.editor). Go here to know how to access the CVS online ;
  6. The next page shows you which tag you could synchronize. Nothing has to be done here, but clicking on the "Next" button will cause an error pop up. You have to get this error. Exit from the error, and cancel the panel. The synchronization needs an other step ;
  7. To import from the CVS, right click on the project, choose Team > Synchronize with Repository. The box of your project should be checked, and only it. Now click on Finish. An information pops up and shows you some statistics. The first time you synchronize your project, you will probably have some conflicts. Go here to understand what to do in this case. Once this step is done, you could update all incoming changes with the button on the top ;
  8. At last, you may need to ignore some files or directories. You can view here how to do that.

The information pop up are a good way to be sure you do not make any mistakes. Do not hide them.

Meaning of CVS icons

When you use a CVS repository, icons in the package explorer will change of look ! To understand the meaning of each icons, you have to go to the CVS Label Decorations documentation page which will show that explicitly.

Resolving conflicts

Lot of people is able to use a file at the same moment ! This will cause a conflict during the synchronization. Three cases could be considered :

  • You do not mind of your version : thus you should update your date from the CVS ;
  • Your data are safe and you are sure you have to upgrade files on the CVS, then you commit your changes ;
  • You do not know what to do, or you are not sure of you may intend on the CVS, so you have to speak with the team which will tell you what you may do !

For example, the first time you catch a project on the CVS, the procedure we encourage you to follow creates a new project with a simple file ".project" at the root. This file is an error for the CVS project, thus you need to update (import) and not the commit (export) this file. You cannot just update a conflicting file, so you have to override and update it. Do so by a right click on the file, and choose Override and update.

Ignoring files

In the CVS update and commit procedures, you may ommit some files or directories. This is simply done in the Team Synchronization perspective with a right click on the file and choose "Add to .cvsignore". The default option proposed is not so bad...

For example, the first time to catch a CVS project, a "bin" directory is locally created, and should not be exported to the CVS. Thus you can add it to the ".cvsignore" file. After this, the file ".cvsignore" will be created, and you can also repeat the procedure with this file.
